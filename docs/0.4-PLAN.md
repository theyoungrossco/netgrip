# 0.4 "Visibility" ‚Äî plan

0.4 makes the container / virtualization layer legible. It is **read-only
clarity first**: show what connects to what before offering any edit. Like 0.2
it lands in milestones that are each independently shippable and testable.

for the why; this file tracks the how and the status.

## Docker (D1 / D2) ‚Äî what's landing now

The host already shows a docker bridge (`docker0` / `br-‚Ä¶`) as an unexplained
bridge and a container's `veth` as an anonymous member. D1/D2 explain it.

### Read (core, no Qt) ‚Äî `probe_docker()`

All best-effort: if `docker` is absent, or the invoking user can't reach the
daemon, the read yields nothing and the rest of the probe is unaffected (same
contract as the wireless / bridge-vlan / DNS reads). Runs unprivileged via
`runner.run`; works locally and over SSH.

- `docker network inspect $(docker network ls -q)` ‚Üí each **network**: name,
  id, driver, the **host bridge ifname** (`Options["com.docker.network.bridge.name"]`,
  e.g. `docker0` for the default `bridge` network, `br-<id12>` for user
  networks), IPAM subnet(s) and gateway.
- `docker ps -q | xargs -r docker inspect` ‚Üí each running **container**: name,
  image, state, **compose project/service** (`com.docker.compose.{project,service}`
  labels), the **networks** it's on with its IP on each
  (`.NetworkSettings.Networks.<net>.IPAddress`), and **published ports**
  (`.NetworkSettings.Ports`: `"80/tcp": [{HostIp, HostPort}]`; `null` = exposed
  but not published, skipped).

Parsed by pure functions (`parse_docker_networks`, `parse_docker_containers`)
into new model dataclasses (`DockerNetwork`, `Container`, `PortMapping`) hung off
`HostState`. The matching host bridge `Interface` is tagged with its docker
network name during enrichment so its box can say so.

### Draw (ui)

- The docker bridge keeps its `GroupNode`, now labelled `docker: <network>`.
- Each container is a **`ContainerNode`** joined by a solid cable to every
  bridge network it's on, showing its image, its **IP per network** and its
  **compose project/service**.
- A container's anonymous host-side `veth` (a bridge member of a docker bridge)
  is **folded away** ‚Äî the container box on that bridge stands for it ‚Äî so
  there's one box per container, not a veth box beside it.
- Published ports draw as a **dashed `PortEdge`** from the container to the
  host's uplink (the interface holding the default route; falls back to the
  bridge). Its `:8080‚Üí80/tcp` label (host-IP-prefixed when not `0.0.0.0`/`::`)
  is **shown only while either end box is selected**, so a busy host isn't a
  wall of port text; when shown it nudges along the line to **dodge boxes**.
- A container-bearing docker network **flows left-to-right from the host uplink**
  (uplink ‚Üí container ‚Üí bridge ‚Üí bridge's IP), via layout-only edges from the
  uplink to each container ‚Äî so a bridge is never stranded in the left column
  (only physical NICs seed it). An **empty** docker bridge (no containers, e.g.
  an unused `docker0`) carries nothing from the host, so it's left to **float**
  as its own island rather than crowding the uplink's column.
- A docker bridge is **titled by its network**: alias if set, else the docker
  network name, else the `br-‚Ä¶` ifname (kept as a detail line).
- **Docker-owned links are read-only** (the bridge and its members):
  `HostState.is_docker_owned` gates the menus and drop handlers, so netgrip
  won't rename/delete/re-address/add-member/move-address on them ‚Äî those go
  through docker / compose.

A new `container` theme colour + glyph + legend row; everything palette-driven
per hard rule 4. Container boxes are movable with remembered positions like any
other box. Read-only: right-clicking a container opens no menu (yet). The peer
resolver also gained a **netns guard** so a container `veth` is never mis-paired
to a host interface whose ifindex its cross-netns peer happens to collide with.

## Known limitations to revisit (Ross will steer)

- **A container's host `veth` is folded away, not merged in detail.** The veth
  box is hidden and the container stands for it, which removes the per-container
  veth noise. Surfacing the *specific* veth's name/MTU on its container box still
  needs the container's `eth0` `iflink` to pin cable‚Üîcontainer 1:1 ‚Äî a netns read
  / `nsenter` (privileged), deferred to **D4**. (Reads are unprivileged today;
  D4 is the point to decide whether that one read may escalate.)
- **Published ports trust docker's own view.** D3 cross-checks against the
  kernel NAT table so a stale/rootless-proxy mapping can't mislead.
- **Compose framing.** Containers show their compose project as a line rather
  than being boxed into a per-project frame; a `ComposeRegion` frame is an easy
  follow-up if the per-container line reads as too loose.
- Only **running** containers are read (exited ones have no veth/IP). `-a` is a
  later toggle if wanted.
